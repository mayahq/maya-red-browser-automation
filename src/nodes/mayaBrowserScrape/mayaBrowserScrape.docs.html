Description

Scrape data from a tab (without necessarily switching to it) using Maya WebQL


Config

query JSON
The WebQL query to execute on the page. Refer to WebQL Documentation to learn more.
timeout number
The time (in milliseconds) within which if the query does not resolve, the node errors out. This property is not functional right now.
tabId number
The unique tab id in which the website to be queried is rendered in. It can be obtained in using {"url":"*://*.[domain]/*"} in the find tab node. For e.g. {"url":"*://*.[domain]/*"} where domain can be anything like google.com or amazon.in. There are other query elements which can be added to same object as referenced above. One common one to be used is for activeTab which takes in a boolean to search for a tab in focus with the url pattern matching above.
session config
Holds the encryption keys used by Maya to securely talk to your browser.

Outputs

The following properties are set on the msg object in case of successful execution.

scrapeResult object
The result of the scraping operation.

On error

The node will set the below-mentioned properties on the msg object, when there is an error

isError boolean
Set to true if the type operation failed for whatever reason. This property might not be set at all in case of no error.
error object
Contains information about the error that occured
