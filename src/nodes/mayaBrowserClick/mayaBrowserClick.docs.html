Description

Click on a specified element on a given (not necessarily open) browser tab.


Config

selectorstring
The xpath for the element to click on. We recommend watching this video to learn more about xpaths (it's slightly long, but its worth it).
timeout number
The amount of time in milliseconds that the node will wait for the target element to load in page. Whenever the element is loaded, the click action is performed immediately. If it doesn't load within this time period, the node will error out.
tabId number
The unique tab id in which the website to be queried is rendered in. It can be obtained in using {"url":"*://*.[domain]/*"} in the find tab node. For e.g. {"url":"*://*.[domain]/*"} where domain can be anything like google.com or amazon.in. There are other query elements which can be added to same object as referenced above. One common one to be used is for activeTab which takes in a boolean to search for a tab in focus with the url pattern matching above.
index number
The index of the element to click on, in case your xpath query resolves to multiple elements. If sure that your query will always resolve to a unique element, just set this to 0.
session config
Holds the encryption keys used by Maya to securely talk to your browser.

Outputs

This node does not set any properties on the msg object in case of successful execution.


On error

The node will set the below-mentioned properties on the msg object, when there is an error

isErrorboolean
Set to true if the click operation failed for whatever reason. This property might not be set at all in case of no error.
error object
Contains information about the error that occured
